# Website Design Packages ‚Äî Design Spec

**Date:** 2026-06-24 ¬∑ **Status:** Approved ‚Üí build ¬∑ **Branch:** `feat/website-packages`

## Goal
Add a purchasable **website design** service at `/websites/` on operateai.com.au.
Three fixed-price tiers, bought via **Stripe Payment Links** (deposit to start),
with a graceful enquiry fallback until the links are configured.

## Positioning (the differentiator)
Not "another web designer" ‚Äî **websites built AI-readable** so ChatGPT / Google AI
can parse a business's services, prices and suburbs and *recommend* it. Plugs into
the existing visibility funnel (free Check ‚Üí done-for-you ‚Üí iOAgent ‚Üí Websites).

## Locked decisions
- **Purchase:** Stripe Payment Links (zero backend, no webhook). Operator notified via Stripe's own payment email/dashboard.
- **Tiers:** 3 (good-better-best), middle featured.
- **Pricing/contents:** drafted as placeholders, edited in one config spot.
- **Payment model:** 50% deposit to start; balance on completion (so the Payment Link charges the deposit).

## Architecture (mirror existing offering pages)
- `web/lib/business.ts` ‚Üí add `WebsitePackage` type + `WEBSITE_PACKAGES` array. Each:
  `{ id, name, tagline, priceFrom, deposit, blurb, features[], timeline, stripeUrl?, featured? }`.
  Placeholder prices + `stripeUrl: ""`. A flagged comment marks both as pre-launch edits.
- `web/lib/business.ts` ‚Üí currency formatting stays in the component (`Intl.NumberFormat` en-AU, AUD, no decimals).
- `web/components/websites/pricing-cards.tsx` (NEW, server component) ‚Äî renders the 3 cards from `WEBSITE_PACKAGES`. Featured card: Signal-tinted border + "Most popular" badge. Each card: name, tagline, big from-price, "Pay $X deposit to start" line, feature ticks, CTA.
  - **CTA logic:** `stripeUrl` set ‚Üí `ButtonLink` href=stripeUrl, label "Start ‚Äî pay $X deposit"; empty ‚Üí href `/contact/`, label "Enquire". (Same no-op-without-config discipline as the rest of the site.)
- `web/app/websites/page.tsx` (NEW, server component) ‚Äî Metadata, JSON-LD (`buildWebPage` + `buildService` + `buildBreadcrumb`; `<Faq>` emits its own). Sections: hero (AI-readable angle, dual CTA) ‚Üí "every site includes" strip ‚Üí `<PricingCards/>` ‚Üí how-it-works (4 steps) ‚Üí FAQ ‚Üí final CTA. Alternating tones, Signal accents, Hanken/JetBrains, no em-dashes in headings.
- `web/lib/nav.ts` ‚Üí add `{ href: "/websites/", label: "Websites" }`.
- `web/app/sitemap.ts` ‚Üí add `/websites/`.
- `web/__tests__/routes.smoke.test.ts` ‚Üí add `{ path: "/websites/", needle: /<hero headline>/i }`.

## CTA note
`ButtonLink` (next/link) renders an `<a>`; external `buy.stripe.com` URLs work as-is.
Stripe checkout opens same-tab (standard). No `target=_blank` needed.

## Out of scope (v1 ‚Äî upgrade path)
Neon order records, automated fulfilment, full Stripe Checkout + webhooks, a custom
post-payment success page (Stripe's confirmation covers it).
